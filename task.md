# 포켓몬 클론 게임 개발 로드맵 (Task List)

## 1. 에셋 및 리소스 검토 (우선순위: 높음)
- [ ] **이미지 에셋 검토**: 캐릭터(플레이어, NPC), 몬스터(포켓몬), 타일셋(맵, 건물, 자연환경), UI 요소(메뉴, 배틀 화면, 대화창) 이미지 파일 확인 및 확보.
- [ ] 사운드 에셋 검토: 배경음악(BGM), 효과음(SFX - 공격, 획득, UI 클릭 등) 확인.
- [ ] 데이터/기획 문서 검토: 속성 상성표, 몬스터 스탯 기획안 등.
- 결과: repo 내 `pixel-editor/assets`에 몬스터 3종(64x64) + 타일 4종(16x16) PNG 존재. 사운드/데이터 에셋 및 공용 assets 폴더는 미정.

## 2. 기획 및 아키텍처 설계
- [ ] 핵심 게임 루프 정의 (탐험 -> 인카운터 -> 전투 -> 포획/성장).
- [ ] 데이터 구조 설계: 몬스터 능력치, 속성, 스킬, 아이템 데이터베이스.
- [ ] 맵 및 레벨 디자인 기획.
- [ ] 기술 스택 선정 및 프로젝트 기본 세팅 (엔진/프레임워크 선택).

## 3. 코어 시스템 구현 (탐험)
- [ ] **이동 시스템**: 타일 기반 2D 그리드 이동 로직 및 애니메이션.
- [ ] **맵 시스템**: 타일맵 렌더링, 충돌 처리(이동 불가 영역, 물 등).
- [ ] **카메라**: 플레이어를 따라다니는 카메라 시스템.

## 4. 전투 시스템 (핵심)
- [ ] 인카운터: 특정 타일(풀숲 등) 이동 시 무작위 확률로 야생 몬스터 조우.
- [ ] 배틀 씬 UI 전환 및 몬스터 등장 연출.
- [ ] 턴제 전투 엔진: 속도 기반 턴 계산, 속성 상성 적용, 데미지 공식.
- [ ] 스킬 처리: 데미지 스킬, 버프/디버프, 상태 이상.
- [ ] 포획 시스템: 포획 확률 계산 로직 및 애니메이션.
- [ ] 전투 종료: 경험치 획득, 레벨업, 스킬 습득.

## 5. UI/UX 및 상호작용
- [ ] 메인 메뉴: 파티(몬스터 목록), 가방(아이템), 플레이어 정보 등.
- [ ] 대화 시스템: NPC 상호작용 시 대화창 출력 (타이핑 효과).
- [ ] 시설 기능: 포켓센터(회복), 상점(아이템 구매/판매) 로직 구현.

## 6. 콘텐츠 연동 및 폴리싱
- [ ] 몬스터/스킬/아이템 대량 데이터 입력.
- [ ] 맵 확장: 여러 마을, 도로, 동굴 제작 및 맵 간 이동 포탈.
- [ ] 트레이너(NPC) 전투 이벤트 구현.
- [ ] 전체적인 애니메이션, 이펙트, 사운드 타이밍 조절.

## 7. 최적화 및 배포
- [ ] 버그 수정 (QA) 및 게임 밸런스 조정.
- [ ] 타겟 플랫폼 빌드 및 배포.
